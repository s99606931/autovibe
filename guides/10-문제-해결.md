## 1. 세션 / 스킬 문제

### 1.1 Claude가 AutoVibe를 모름

**증상**: Claude가 `/av` 명령어나 AutoVibe 관련 요청을 이해하지 못함

**원인**: CLAUDE.md에 AutoVibe 섹션이 없거나, `.claude/` 디렉토리가 없음

**해결**:

```bash
# 1. .claude/ 디렉토리 확인
ls -la .claude/

# 2. CLAUDE.md 확인
grep "AutoVibe" CLAUDE.md
```

디렉토리가 없으면:
```
docs/autovibe/design/av-ecosystem-design-spec.md 파일을 읽고
AutoVibe Phase 0부터 구축해줘.
```

CLAUDE.md에 AutoVibe 섹션이 없으면:
```
/av-base-sync
```

### 1.2 /av 명령어가 안 됨

**증상**: `/av`를 입력해도 스킬이 호출되지 않음

**원인**: `.claude/skills/av/SKILL.md` 파일이 없거나 손상됨

**해결**:

```bash
# 1. 스킬 파일 확인
ls .claude/skills/av/SKILL.md

# 2. 파일이 없으면 Phase 5 재실행
```

Claude에게:
```
.claude/skills/av/SKILL.md 파일이 없어.
docs/autovibe/design/av-ecosystem-design-spec.md의 5.2 섹션을 참고해서 재생성해줘.
```

## 2. 에이전트 문제

### 2.1 PM이 대화를 시작하지 않음

**증상**: `/av {기능}` 입력 후 PM 대화가 시작되지 않고 바로 코드 생성

**원인**: ROUTING_TABLE에서 PM 라우팅이 안 됨, 또는 에이전트 파일 누락

**해결**:

```bash
# 1. PM 에이전트 파일 확인
ls .claude/agents/av-pm-coordinator.md

# 2. ROUTING_TABLE 확인
grep -A 5 "pm + feature" .claude/skills/av/SKILL.md
```

PM 에이전트가 없으면:
```
av-pm-coordinator.md 에이전트를 생성해줘.
docs/autovibe/design/av-ecosystem-design-spec.md의 4.2 섹션을 참고해서.
```

### 2.2 Agent Team 스폰 실패

**증상**: PL이 Agent Team을 만들지 못함

**원인**: Agent Teams 기능이 비활성화 상태

**해결**:

```bash
# settings.json 확인
cat .claude/settings.json | grep AGENT_TEAMS
```

없으면 추가:
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

Claude Code 재시작 후 다시 시도합니다.

## 3. 플러그인 문제

### 3.1 gstack이 동작하지 않음

**증상**: 브라우저 테스트, 스크린샷 명령이 실패

**원인**: gstack 플러그인 미설치 또는 브라우저 환경 문제

**해결**:

```bash
# 1. gstack 확인
claude
/gstack
```

gstack 메뉴가 나타나지 않으면:
```
/plugin install gstack
```

WSL 환경에서 브라우저 문제:
```bash
# Chrome/Chromium 설치 확인
which google-chrome || which chromium-browser

# 없으면 설치
sudo apt install -y chromium-browser
```

### 3.1.1 gstack navigate 타임아웃

**증상**: `Skill("gstack", "navigate localhost:3000")` 실행 시 타임아웃

**원인**: 개발 서버가 실행되지 않았거나, 포트가 다름

**해결**:

```bash
# 1. 개발 서버 실행 확인
curl -s http://localhost:3000 | head -5

# 2. 실행 중인 포트 확인
lsof -i -P -n | grep LISTEN | grep node

# 3. 서버 실행 후 재시도
npm run dev &
```

### 3.1.2 gstack 스크린샷이 빈 화면

**증상**: 스크린샷이 캡처되지만 빈 흰 화면

**원인**: 페이지 로드 완료 전 캡처, 또는 CSR 앱에서 JS 실행 대기 부족

**해결**:

```
# gstack에 대기 시간 추가
/gstack navigate http://localhost:3000 --wait 3000
```

### 3.1.3 gstack + bkit 연동 실패

**증상**: `/av-base-post-qa` 실행 시 gstack은 성공하지만 bkit:qa-monitor 실패

**원인**: Docker 컨테이너가 실행되지 않음 (bkit:qa-monitor는 Docker 로그 기반)

**해결**:

```bash
# Docker 실행 확인
docker ps

# Docker 없이 QA 하려면 gstack만 사용
/gstack navigate http://localhost:3000
/gstack check-errors
```

### 3.2 bkit이 동작하지 않음

**증상**: PDCA 문서 작성, 갭 분석 등이 실패

**원인**: bkit 플러그인 미설치

**해결**:

```bash
claude
/bkit
```

bkit 메뉴가 나타나지 않으면:
```
# Step 1: Marketplace 등록
/plugin marketplace add popup-studio-ai/bkit-claude-code

# Step 2: 설치
/plugin install bkit

# Step 3: 확인
/bkit
```

## 4. 인프라 문제

### 4.1 Registry 불일치

**증상**: `/av-vibe-forge health`에서 Registry 불일치 경고

**원인**: Forge를 거치지 않고 파일을 직접 추가/삭제

**해결**:

```
# Registry 재생성
.claude/ 디렉토리의 모든 컴포넌트를 스캔해서
components.json을 재생성해줘.
```

또는:
```
/av-vibe-forge health --fix
```

### 4.2 메모리가 로드되지 않음

**증상**: 에이전트가 이전 세션의 학습 내용을 기억하지 못함

**원인**: `memory: project` 필드 누락 또는 메모리 파일 손상

**해결**:

```bash
# 1. Agent frontmatter 확인
head -20 .claude/agents/av-pm-coordinator.md | grep memory

# 2. 메모리 디렉토리 확인
ls .claude/agent-memory/av-pm-coordinator/
```

`memory: project` 필드가 없으면 frontmatter에 추가:
```yaml
memory: project
```

메모리 디렉토리가 없으면 → 에이전트를 한 번 실행하면 자동 생성됩니다.

### 4.3 훅이 실행되지 않음

**증상**: 파일 작성 후 자동 감사가 안 됨, 세션 시작 대시보드가 안 나옴

**원인**: settings.json에 훅이 등록되지 않았거나, 셸 스크립트 실행 권한 없음

**해결**:

```bash
# 1. settings.json 훅 등록 확인
cat .claude/settings.json | grep hooks -A 20

# 2. 셸 스크립트 실행 권한 확인
ls -la .claude/hooks/

# 3. 실행 권한 부여
chmod +x .claude/hooks/*.sh
```

settings.json에 훅이 없으면:
```
settings.json에 훅을 등록해줘.
docs/autovibe/design/av-ecosystem-design-spec.md의 훅 섹션을 참고해서.
```

## 5. 성능 문제

### 5.1 토큰 사용량이 너무 많음

**증상**: 세션 시작이 느리거나, 컨텍스트 초과 경고

**원인**: 모든 Rule/Skill이 항상 로드됨 (paths 미설정)

**해결**:

```
/av-ecosystem-optimizer
```

수동 최적화:

```yaml
# Before: 항상 로드
---
name: av-base-spec
---

# After: 지연 로딩 (.claude 파일 열 때만)
---
name: av-base-spec
paths:
  - ".claude/**"
---
```

### 5.2 세션이 느려짐

**증상**: 2시간 이상 사용 후 응답이 느려짐

**원인**: 긴 세션에서 컨텍스트 누적

**해결**:

```bash
# Claude Code 세션 리프레시
/clear
```

또는 Claude Code를 종료 후 재시작:
```bash
exit
claude
```

Agent Teams 장시간 사용 시:
- 2시간 이상 세션은 `/clear`로 주기적 정리 권장
- 백그라운드 에이전트가 남아있으면 `Ctrl+F`로 정리

## 6. 3축 통합 문제

### 6.1 3축 중 하나가 없을 때의 대안

AutoVibe 3축(Claude Code, gstack, bkit) 중 일부를 사용할 수 없을 때:

| 상황 | 영향 | 대안 |
|------|------|------|
| **gstack 미설치** | 브라우저 검증 불가 | 수동으로 브라우저 확인 후 `/pdca status`로 진행 |
| **bkit 미설치** | PDCA 문서·Gap 분석 불가 | `docs/` 디렉토리에 수동 문서 작성, 코드 리뷰로 대체 |
| **gstack + bkit 모두 없음** | Claude Code만으로 동작 | 코드 생성은 가능하나 품질 보증 사이클 불가 — 플러그인 설치 권장 |

### 6.2 PDCA 사이클이 진행되지 않음

**증상**: `/av {기능}` 입력 후 PM 대화는 하지만 Plan/Design이 생성되지 않음

**원인**: bkit:pdca 스킬이 로드되지 않음

**해결**:

```
# bkit PDCA 직접 확인
/bkit:pdca status

# 안 되면 bkit 재설치
/plugin install bkit
```

### 6.3 Gap 분석 후 자동 개선이 안 됨

**증상**: Match Rate < 90%인데 자동 반복이 시작되지 않음

**원인**: pdca-iterator 에이전트가 호출되지 않음

**해결**:

```
# 수동으로 반복 개선 시작
/pdca iterate {feature}
```
