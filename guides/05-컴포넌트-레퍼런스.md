# 05. 컴포넌트 레퍼런스

> **목표**: Rule, Agent, Skill, Hook 4가지 컴포넌트의 상세 스펙을 레퍼런스로 활용합니다.
> **용도**: 필요할 때 찾아보는 참고 문서

---

## 컴포넌트 요약 (av 본 프로젝트 기준, 6차 PDCA 2026-05-13)

| 유형 | 수량 | 파일 위치 | 호출 방식 |
|------|------|----------|----------|
| **Rule** | 7종 | `.claude/rules/*.md` | 자동 로드 / paths 지연 로딩 |
| **Agent** | 13종 | `.claude/agents/*.md` | `Agent()` (Task는 별칭) |
| **Skill** | 19종 | `.claude/skills/*/SKILL.md` | `/스킬명` / `Skill()` |
| **Hook** | 12종 | `.claude/hooks/*.sh` | Claude Code 이벤트 자동 |

> 위는 av 마스터 프로젝트의 현재 누적치. 신규 프로젝트는 Phase 0~6 단계별로 부분 생성.

신규 컴포넌트 (6차 PDCA):
- Rule: `av-base-task-lock.md` — 멀티 세션 작업 락 정책
- Skill: `av-base-task-lock`, `av-base-codegraph` — 락 게이트웨이, 코드 그래프 게이트웨이
- Hook: `av-session-id-init.sh` (SessionStart), `av-pm-team-lock-check.sh` (UserPromptSubmit)

## 1. Rule (규칙)

### Frontmatter 형식

```yaml
---
name: av-{rule-name}
autovibe: true
version: "1.0"
created: "YYYY-MM-DD"
group: base|vibe|util|{domain}
paths:                              # 선택 — 지연 로딩
  - "{glob-pattern}"
---
```

### 필드 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `name` | O | `av-` 접두사 + kebab-case |
| `autovibe` | O | 항상 `true` |
| `version` | O | 시맨틱 버전 |
| `group` | O | base/vibe/util/{domain} |
| `paths` | X | 지연 로딩 glob 패턴 — 없으면 항상 로드 |

### 전체 Rule 목록

| Rule | 그룹 | paths | 역할 |
|------|------|-------|------|
| `av-base-spec` | base | `.claude/agents/**`, `.claude/skills/**`, `.claude/rules/**` | 중앙 규칙 인덱스 |
| `av-org-protocol` | base | — (항상 로드) | PM→PL→Agent 승인 프로토콜 |
| `av-base-memory-first` | base | `.claude/**` | 메모리 우선 읽기 원칙 |
| `av-util-mermaid-std` | util | — | Mermaid 다이어그램 표준 |
| `av-base-plugin-routing` | base | — (항상 로드) | gstack/bkit 플러그인 라우팅 |

## 2. Agent (에이전트)

### Frontmatter 형식

```yaml
---
# 공식 필수
name: av-{agent-name}
description: |
  역할 설명 (1~3줄)
  트리거: 언제 호출되는지

# 공식 권장
tools: [Read, Glob, Grep, Write, Edit]
disallowedTools: []
model: sonnet|opus|haiku
permissionMode: default|plan|dontAsk
maxTurns: 50
memory: project|user|local
background: false
effort: medium|high|max

# av 커스텀
autovibe: true
version: "1.0"
created: "YYYY-MM-DD"
group: base|vibe|{domain}
---
```

### 필드 설명

| 필드 | 필수 | 기본값 | 설명 |
|------|------|--------|------|
| `name` | O | — | 에이전트 고유 이름 |
| `description` | O | — | 역할 + 트리거 설명 |
| `tools` | 권장 | 전체 | 사용 허용 도구 |
| `disallowedTools` | 권장 | [] | 사용 금지 도구 |
| `model` | 권장 | inherit | sonnet/opus/haiku |
| `permissionMode` | 권장 | default | 권한 처리 모드 |
| `maxTurns` | 권장 | 50 | 최대 에이전틱 턴 |
| `memory` | 권장 | — | project: 영구 메모리 자동 관리 |
| `background` | X | false | 백그라운드 실행 |
| `effort` | X | medium | 추론 노력 수준 |

## 3. Skill (스킬)

### Frontmatter 형식

```yaml
---
# 공식 필수
name: av-{skill-name}
description: |
  스킬 역할 설명

# 공식 권장
argument-hint: "{서브커맨드 힌트}"
user-invocable: true
allowed-tools: [Read, Write, Edit, Glob, Grep, Bash]
context: fork                       # 격리 실행
model: sonnet|opus
effort: medium|high|max
paths: ["src/**/*.ts"]              # 지연 로딩

# av 커스텀
autovibe: true
version: "1.0"
created: "YYYY-MM-DD"
group: base|vibe|{domain}
tier: meta|null
---
```

### 핵심 필드 설명

| 필드 | 설명 |
|------|------|
| `user-invocable` | true면 사용자가 `/스킬명`으로 직접 호출 가능 |
| `context: fork` | 격리된 서브에이전트에서 실행 (메인 컨텍스트 보호) |
| `argument-hint` | `/스킬명` 뒤에 올 인자 힌트 |
| `paths` | 매칭 파일 열 때만 스킬 로드 (컨텍스트 절약) |

### 문자열 치환 (공식 지원)

| 치환 | 설명 |
|------|------|
| `$ARGUMENTS` | 전체 인자 문자열 |
| `$ARGUMENTS[0]`, `$1` | 첫 번째 인자 |
| `${CLAUDE_SESSION_ID}` | 현재 세션 ID |
| `${CLAUDE_SKILL_DIR}` | 스킬 디렉토리 경로 |
| `` !`command` `` | 전처리 명령어 실행 결과 |

## 4. Hook (훅)

### 이벤트 유형

| 이벤트 | 발생 시점 | 예시 |
|--------|---------|------|
| `PreToolUse` | 도구 실행 전 | 파일 쓰기 전 경고 |
| `PostToolUse` | 도구 실행 후 | 파일 쓰기 후 감사 |
| `SessionStart` | 세션 시작 시 | 상태 대시보드 표시 |
| `SubagentStart` | 에이전트 시작 시 | 에이전트 로깅 |
| `SubagentStop` | 에이전트 종료 시 | 결과 로깅 |
| `ConfigChange` | 설정 변경 시 | 설정 백업 |

### 훅 파일 형식

```bash
#!/bin/bash
# Hook: av-post-write-monitor
# Event: PostToolUse
# Trigger: Write, Edit

# stdin으로 이벤트 데이터 (JSON) 수신
EVENT_DATA=$(cat)
TOOL_NAME=$(echo "$EVENT_DATA" | jq -r '.tool_name')
FILE_PATH=$(echo "$EVENT_DATA" | jq -r '.tool_input.file_path // empty')

# 로직 수행
if [[ "$FILE_PATH" == *.md ]]; then
    echo "WARN: Documentation file modified: $FILE_PATH"
fi
```

### 전체 Hook 목록

| Hook | 이벤트 | 역할 |
|------|--------|------|
| `av-post-write-monitor` | PostToolUse(Write, Edit) | 파일 변경 감지 + 감사 |
| `av-session-discovery` | SessionStart | 세션 시작 대시보드 |
| `av-agent-spawn-logger` | SubagentStart | 에이전트 시작 로깅 |
| `av-agent-complete-logger` | SubagentStop | 에이전트 종료 + 결과 로깅 |
| `av-config-watcher` | ConfigChange | 설정 변경 감지 + 백업 |
| + 3개 프로젝트 커스텀 | Phase 5에서 대화로 결정 | — |

### settings.json 훅 등록

```json

    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": ".claude/hooks/av-post-write-monitor.sh"
      }
    ],
    "SessionStart": [
      {
        "command": ".claude/hooks/av-session-discovery.sh"
      }
    ],
    "SubagentStart": [
      {
        "command": ".claude/hooks/av-agent-spawn-logger.sh"
      }
    ]
  }
}
```
