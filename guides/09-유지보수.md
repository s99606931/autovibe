## 1. 매일: 상태 확인

### 세션 시작 시 자동

```
┌─── Workflow Map ─────────────────────────────────┐
│ [PM ✓]→[PLAN ✓]→[DESIGN ✓]→[DO ▶]→[CHECK ·]    │
│ Iter: 0  •  matchRate: N/A                       │
└──────────────────────────────────────────────────┘
```

### 수동 확인

```
/pdca status
```

---

## 2. 매주: 건강도 + 동기화

### 건강도 확인

```
/av-vibe-forge health
```

확인 항목:
- 모든 컴포넌트가 Registry에 등록되어 있는가
- 파일이 존재하는가 (삭제된 파일 참조 없는지)
- frontmatter 필수 필드가 누락되지 않았는가

### CLAUDE.md 동기화

```
/av-base-sync
```

컴포넌트가 추가/삭제되었을 때 CLAUDE.md의 AutoVibe 섹션을 자동 갱신합니다.

---

## 3. 격주: 최적화

### 생태계 최적화

```
/av-ecosystem-optimizer
```

분석 항목:

| 항목 | 검사 내용 | 조치 |
|------|---------|------|
| **토큰 사용량** | 전체 컴포넌트의 토큰 합계 | paths 지연 로딩 추가 |
| **중복 컴포넌트** | 기능 겹치는 에이전트/스킬 | 통합 또는 제거 |
| **미사용 컴포넌트** | 한 달간 호출 0회 | deprecated 처리 |
| **메모리 크기** | 에이전트 메모리 크기 | 오래된 항목 정리 |

### 토큰 절약 전략

```yaml
# Before: 항상 로드 (토큰 소비)
---
name: av-base-spec
---

# After: 지연 로딩 (필요할 때만)
---
name: av-base-spec
paths:
  - ".claude/agents/**"
  - ".claude/skills/**"
---
```

## 4. 매월: 메모리 정리 + 백업

### 메모리 정리

```
/av-ecosystem-optimizer trim-memory
```

에이전트 메모리에서:
- 3개월 이상 된 실행 이력 → 요약 후 삭제
- 더 이상 유효하지 않은 학습 패턴 → 삭제
- 중복 기록 → 통합

### 백업

```bash
# .claude/ 전체 백업
tar -czf autovibe-backup-$(date +%Y%m%d).tar.gz .claude/

# 또는 git commit으로 버전 관리
git add .claude/
git commit -m "chore: AutoVibe 생태계 월간 백업"
```

### 메모리 크기 기준

| 크기 | 상태 | 조치 |
|------|------|------|
| < 10KB | 정상 | 유지 |
| 10~50KB | 주의 | 격주 정리 |
| > 50KB | 위험 | 즉시 정리 |

## 건강도 지표 대시보드

### 전체 지표

| 지표 | 정상 | 주의 | 위험 | 확인 방법 |
|------|------|------|------|----------|
| Registry 동기화 | Synced | 1~2 불일치 | 3+ 불일치 | `/av-vibe-forge health` |
| 컴포넌트 파일 | 전부 존재 | 1개 누락 | 2+ 누락 | `/av-vibe-forge health` |
| 메모리 크기 | < 10KB | 10~50KB | > 50KB | `/av-ecosystem-optimizer` |
| 토큰 사용량 | < 50K | 50~100K | > 100K | `/av-ecosystem-optimizer` |
| 미사용 컴포넌트 | 0 | 1~3개 | 5+ | `/av-ecosystem-optimizer` |
| CLAUDE.md 정합성 | 일치 | — | 불일치 | `/av-base-sync` |

### 자동 감지

`av-post-write-monitor` 훅이 `.claude/` 파일 변경을 감지하면 자동으로 정합성을 확인합니다.

## 문제 예방 패턴

### 패턴 1: 컴포넌트 생성은 반드시 Forge를 통해

```
# 올바름 — Registry 자동 등록
/av-vibe-forge agent payment-lead --group payment

# 잘못됨 — Registry 불일치 발생
# .claude/agents/av-payment-lead.md를 직접 생성
```

### 패턴 2: 삭제도 Forge를 통해

```
# 올바름 — Registry 자동 정리
/av-vibe-forge remove agent payment-lead

# 잘못됨 — 고아 Registry 항목
# rm .claude/agents/av-payment-lead.md
```

### 패턴 3: 설정 변경 후 동기화

```
# settings.json 직접 수정 후
/av-base-sync
```

### 패턴 4: 대규모 변경 후 건강도 확인

```
# 여러 컴포넌트 변경 후
/av-vibe-forge health
```

## 장기 운영 팁

### 1. Git으로 생태계 버전 관리

`.claude/` 디렉토리를 git에 포함하면 생태계 변경 이력을 추적할 수 있습니다.

```gitignore
# .gitignore — 메모리는 제외 (개인 데이터)
.claude/agent-memory/
```

### 2. Claude Code 업데이트 시

```bash
claude --version    # 현재 버전
claude update       # 업데이트
```

업데이트 후 `/av-vibe-forge health`로 호환성을 확인합니다.

### 3. 팀 공유

- `.claude/rules/`, `.claude/agents/`, `.claude/skills/`, `.claude/hooks/` → git 공유
- `.claude/agent-memory/` → 개인별 (git 제외)
- `.claude/registry/components.json` → git 공유

### 4. 성능 모니터링

gstack benchmark 결과를 Memory Keeper가 축적합니다:

```
/av-agent-chat av-base-memory-keeper
> 최근 성능 기준선 보여줘
```
